Mini API Node com SQLite

Esta aplicação expõe dois conjuntos de endpoints REST em /api/users e /api/orders. Todos aceitam e devolvem JSON.

1. Users

  • GET /api/users – lista todos os usuários.
  • POST /api/users – cria usuário.
    Exemplo de corpo: {"name":"Ana","email":"ana@exemplo.com"}
  • GET /api/users/:id – detalhe do usuário.
  • PUT /api/users/:id – atualiza usuário.
  • DELETE /api/users/:id – remove usuário.
  • GET /api/users-totals – join que devolve cada usuário com a soma de preços de seus pedidos.

2. Orders

  • GET /api/orders – lista todos os pedidos.
  • POST /api/orders – cria pedido.
    Exemplo: {"user_id":1,"product":"Livro","price":49.90}
  • GET /api/orders/:id – detalhe do pedido.
  • PUT /api/orders/:id – atualiza pedido.
  • DELETE /api/orders/:id – remove pedido.
  • GET /api/orders-with-user – join que traz cada pedido com nome e e-mail do usuário associado.

Exemplos de uso via curl

curl -X POST http://localhost:3000/api/users \
     -H "Content-Type: application/json" \
     -d '{"name":"Ana","email":"ana@exemplo.com"}'

curl http://localhost:3000/api/users-totals
